Built as a portfolio-ready submission for **Data Analyst / Data Science
internship applications**.

---

## 📌 Project Description

Retail businesses generate large volumes of transactional data — but raw data
alone tells no story. **Smart Retail Sales Analysis** turns a messy,
real-world-style retail dataset into actionable insight through a complete
analytics pipeline:

1. **Data Cleaning** — handling missing values, duplicates, inconsistent text,
   mixed date formats, and outliers.
2. **Exploratory Data Analysis (EDA)** — trend analysis, category/region
   breakdowns, correlation analysis, and distribution checks.
3. **Machine Learning** — predicting order-level Sales using **Linear
   Regression** and **Random Forest Regressor**, with full performance
   comparison.
4. **Interactive Dashboard** — a Streamlit web app with KPI cards, filters,
   multi-tab analysis views, a live prediction tool, and a downloadable report.

---

## ✨ Features

### 📊 Data Analysis
- Automated data cleaning pipeline (missing values, duplicates, dtype fixes)
- Outlier detection using the IQR method (with winsorization, not deletion)
- Summary statistics and grouped aggregations
- Feature engineering (Year, Month, Profit Margin %)

### 📈 Visualizations
- Monthly Sales & Profit Trends
- Category-wise and Sub-Category-wise Sales
- Region-wise Sales & Profit
- Customer Segment breakdown
- Correlation Heatmap
- Top 10 Best-Selling Products
- Sales Distribution histogram

### 🤖 Machine Learning
- **Linear Regression** (interpretable baseline)
- **Random Forest Regressor** (non-linear, higher accuracy)
- Evaluated with **MAE**, **RMSE**, and **R² Score**
- Automatic best-model selection, saved with `joblib` for the dashboard

### 🖥️ Streamlit Dashboard
- Clean, modern UI with a custom teal & coral theme
- 5 live KPI cards (Sales, Profit, Orders, AOV, Profit Margin)
- Sidebar filters: Date range, Region, Category, Customer Segment
- 6 analysis tabs: Sales, Profit, Regional, Category, Prediction, Report
- Interactive **Sales Prediction** tool powered by the trained model
- One-click **CSV report download** (summary + category/region breakdown)

## 🧬 Dataset

The project uses a retail sales dataset with the following fields:

| Column             | Description                                  |
|---------------------|-----------------------------------------------|
| Order ID            | Unique order identifier                       |
| Order Date          | Date the order was placed                     |
| Category            | Product category (Furniture, Technology, etc.)|
| Sub-Category        | Product sub-category                          |
| Product Name        | Name of the product sold                      |
| Region              | Sales region (North, South, East, West, Central) |
| Customer Segment    | Consumer / Corporate / Home Office            |
| Quantity            | Units sold per order                          |
| Discount            | Discount applied to the order                 |
| Sales               | Order revenue ($)                             |
| Profit              | Order profit ($)                              |

> **Note:** `data/retail_sales_raw.csv` is a synthetically generated dataset
> (~9,950 rows spanning 2021–2024) designed to mirror the structure of the
> well-known "Superstore" retail dataset, with deliberately injected missing
> values, duplicate rows, mixed date formats, and outliers — so the cleaning
> pipeline has genuine, realistic work to do. Swap in your own retail CSV
> (with matching column names) to analyze real data.

## 🚀 Usage Guide

### Step 1 — Run the Data Cleaning & EDA notebook
```bash
cd notebooks
python 01_data_cleaning_eda.py
```
This cleans `data/retail_sales_raw.csv`, saves the cleaned dataset to
`data/retail_sales_clean.csv`, and exports all charts to `/images`.

### Step 2 — Train the Machine Learning models
```bash
python 02_sales_prediction_model.py
```
This trains Linear Regression and Random Forest models, evaluates both,
saves the best-performing model to `/models/best_sales_model.pkl`, and
exports a model comparison chart.

> 💡 Tip: both scripts are written in `# %%` cell-block style, so you can
> also open them directly in **Jupyter** (via Jupytext) or **VS Code**
> as interactive notebooks.

### Step 3 — Launch the Streamlit Dashboard
```bash
cd ..
streamlit run app.py
```
Then open the URL shown in your terminal (typically `http://localhost:8501`).

## 📈 Model Performance

| Model              | MAE     | RMSE    | R² Score |
|--------------------|---------|---------|----------|
| Linear Regression  | ~943    | ~1270   | ~0.71    |
| **Random Forest**  | **~534**| **~932**| **~0.84**|

**Random Forest Regressor** was selected as the best model — it captures
non-linear relationships between Quantity, Discount, Category, and Region
that the linear model misses, cutting average prediction error by ~43%.

*(Exact values will vary slightly between runs/dataset versions — see
`models/model_comparison.csv` after training for your run's actual numbers.)*

## ☁️ Deployment

### Deploy to Streamlit Community Cloud

1. Push this project to a **public GitHub repository** (see steps below).
2. Go to [share.streamlit.io](https://share.streamlit.io) and sign in with GitHub.
3. Click **"New app"** → select your repository, branch (`main`), and set
   the main file path to `app.py`.
4. Click **Deploy**. Streamlit Cloud will install `requirements.txt`
   automatically and launch your dashboard with a public URL.

> ⚠️ Make sure `data/retail_sales_clean.csv` and `models/best_sales_model.pkl`
> are committed to the repo (or regenerate them via a startup script), since
> Streamlit Cloud only has access to what's in your GitHub repo.

### Push the project to GitHub

```bash
cd Smart-Retail-Sales-Analysis
git init
git add .
git commit -m "Initial commit: Smart Retail Sales Analysis project"
git branch -M main
git remote add origin https://github.com/<your-username>/Smart-Retail-Sales-Analysis.git
git push -u origin main
```
